Accessibility Presentation and Testing of My Website

Accessibility is a critical aspect of modern web development, ensuring that websites are usable for everyone, including people with disabilities. While automated tools can help identify and address accessibility issues, they are not exhaustive. This report presents the accessibility choices made in the development of my site, including insights from automated testing and additional manual inspections to ensure compliance with universal design principles.

Automated Accessibility Testing

I tested the site using Lighthouse, an automated tool that evaluates various aspects of accessibility, such as contrast, semantic structure, and proper use of attributes. The tools flagged the following strengths and areas for improvement:

Strengths

  • Alt Attributes: All images included descriptive alt text, ensuring they are accessible to screen readers. For example, the Audi image uses the alt text: "Audi," which briefly describes the car for visually impaired users.
  • Color Contrast: The siteâ€™s color scheme was tested to be colorblind-friendly, with high contrast ratios between text and background.
  • Semantic Structure: Proper HTML5 elements, such as <article>, <figure>, and <figcaption>, were used for content organization, aiding assistive technologies in navigating the site.

Areas for Improvement

  • Oversized Images: Oversized images were identified in the initial design, leading to inefficiencies in loading. This issue was resolved by implementing responsive images.

Screenshots from the automated tests show a strong performance in accessibility categories. These tests provided a solid foundation for ensuring universal design compliance.

Manual Accessibility Checks

Automated tools cannot catch everything. We conducted manual checks for the following:

  1. Logical Tab Order: Navigation via the keyboard was tested to ensure all interactive elements, such as links and buttons, followed a logical tab order. For instance, the <a> tags linking to the Audi official site and other external resources were tested for focus and hover states.
  2. Responsive Design: Media queries and flexible layouts were used to ensure the site adapts seamlessly across devices. For instance, the .collaboration container dynamically scales with max-width and padding defined in percentages to prevent horizontal scrolling or content overflow.
  3. Image Optimization: To reduce bandwidth usage and improve loading speeds, we used smaller images, avoiding the inefficiency of large images being displayed smaller.
  4. Hover and Focus States: Interactive elements, like images, were manually checked to ensure they provide visual feedback when focused or hovered, enhancing usability for keyboard users.

Conclusion

While automated tests are a helpful starting point, manual testing revealed nuances that require careful coding decisions to create a fully accessible site. Semantic HTML, responsive design, optimized images, and careful attention to interactivity ensure the site is inclusive for all users.
